# Changelog

All notable changes to **Solva** will be documented in this file.

This project uses a simple, human-readable changelog during early development.

---

## v1.1 — Cleaning-First Rebuild (In Progress)

### Added
- Clear positioning of Solva as a scheduling tool for **solo cleaning businesses**
- New repository structure separating:
  - Public marketing site
  - Active application
  - Archived prototype
- Defined product scope and intentional non-goals
- Public-facing README aligned to product intent

### Changed
- Reduced feature scope to core daily scheduling workflows
- Shift from exploratory prototype patterns to a maintainable foundation
- Reframed the product around a single, well-defined service use case

### Removed
- Prototype-only UI patterns that added unnecessary complexity
- Early feature experiments not aligned with daily solo use
- Assumptions around payments, teams, and heavy automation

---

## v1.0 — Prototype (Archived)

### Summary
Initial exploratory prototype used to test ideas, flows, and layout concepts.

### Included
- Multiple scheduling and booking flows
- Early dashboard concepts
- Experimental UI and layout patterns

### Outcome
- Valuable feedback gathered
- Product direction clarified
- Prototype frozen and archived for reference
